OC基础之NSTimer和CADisplayLink

Mar 25, 2015

不同的编程语言在构造软件系统的时候都会遇到定时执行某项Task的需求，这就涉及到timer的使用！

iOS系统中提供了NSTimer和CADisplayLink供开发者执行定时任务

NSTimer

“当你设置一个NSTimer，他会被插入到当前任务列表中，然后直到指定时间过去之后才会被执行。但是何时启动定时器并没有一个时间上限，而且它只会在列表中上一个任务完成之后开始执行。这通常会导致有几毫秒的延迟，但是如果上一个任务过了很久才完成就会导致延迟很长一段时间。”

上述问题就是NSTimer的缺点，在执行时间粒度要求不高的任务时十分方便，但是具体到时间粒度要求较高的Task,尤其是执行动画时就显得吃力了；

NSTimer使用上需要注意的问题
  • 由于NSRunloop的模式设置，NSTimer在设置为NSDefaultRunLoopMode时，页面滚动会限制timer中Task的执行，原因是由于页面滚动后NSRunloop的模式为UITrackingRunLoopMode优先级更高；如果想消除对NSTimer执行task的影响可以将task的执行单独放在另外的线程中；
  • 使用NSTimer要注意内存泄露问题；在timer不再使用要记得销毁timer；

CADisplayLink

比起NSTimer，CADisplayLink可以确保系统渲染每一帧的时候我们的方法都被调用，从而保证了动画的流畅性。 

self.timer = [CADisplayLink displayLinkWithTarget:self
                                         selector:@selector(step:)];
[self.timer addToRunLoop:[NSRunLoop mainRunLoop]forMode:NSDefaultRunLoopMode];

Run Loop 模式

为了保证用户界面保持平滑，iOS会提供和用户界面相关任务的优先级，而且当UI很活跃的时候的确会暂停一些别的任务

  • NSDefaultRunLoopMode - 标准优先级
  • NSRunLoopCommonModes - 高优先级
  • UITrackingRunLoopMode - 用于UIScrollView和别的控件的动画
