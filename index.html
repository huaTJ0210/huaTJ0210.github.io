Objective-C之分类

发表于 2017-04-21 | 分类于 Objective-C

在实际的开发过程中，经常会遇到给现有的系统类增加新方法的需求。根据面向对象语言的特性可以使用继承来实现。但遇到给系统类（NSString、NSArray等）新增方法或者按照功能拆分大型类文件时，就需要使用分类这一技术了。在Objective-C语言中分类特性是一种利用runtime的手段，可以满足给现有类直接新增方法的需求。具体使用是新建Category文件，这样在新建在Category的文件中就可以添加方法（实例方法、类方法）、协议、属性;其中属性的实现还需要根据关联对象来辅助才能实现；

分类的用途

  • 声明私有方法:在当前类的.m文件中声明并实现，对外不进行暴露
  • 分解体积庞大的类文件[按照功能模块拆分自定义类或者为系统类增加方法]
  • 公开frameWork中的一些私有方法【较少使用】

Objective-C之KVC

发表于 2017-02-27 | 分类于 Objective-C

什么是KVC

key value coding：是一种使用字符串标识符间接访问对象属性的机制； 

NSObject 提供如下方法：
     [obj setValue:<#(nullable id)#> forKey:<#(nonnull NSString *)#>];
     [obj setValue:<#(nullable id)#> forKeyPath:<#(nonnull NSString *)#>];
     [obj setValue:<#(nullable id)#> forUndefinedKey:<#(nonnull NSString *)#>];
     [obj setValuesForKeysWithDictionary:<#(nonnull NSDictionary<NSString *,id> *)#>];
     [obj valueForKey:<#(nonnull NSString *)#>];
     [obj valueForKeyPath:<#(nonnull NSString *)#>];
     [obj valueForUndefinedKey:<#(nonnull NSString *)#>];

如何使用KVC

KVCObjectTest* kvcTest = [KVCObjectTest new];
       // setvalue
[kvcTest setValue:@"hua" forKey:@"name"];
       // getvalue
 NSLog(@"name:%@",[kvcTest valueForKey:@"name"]);

Objective-C之KVO

发表于 2017-02-20 | 分类于 Objective-C

什么是KVO

KVO,即：Key-Value Observing，它提供一种机制，当指定的对象的属性被修改后，则对象就会接收到通知。简单的说就是每次指定的被观察的对象的属性被修改后，KVO就会自动通知相应的观察者了。

  • KVO是Objective-C对观察者设计模式的又一实现
  • Apple使用了isa混写（isa-swizzling）来实现KVO
