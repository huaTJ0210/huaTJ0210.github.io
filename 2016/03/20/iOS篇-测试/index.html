iOS篇-测试

发表于 2016-03-20 | 分类于 iOS

Xcode中集成XCTest框架进行简单的单元测试；在创建App工程时默认创建UnitTest和UITest；UnitTest可以很方便的自定义Method验证项目中某一个业务逻辑单元，UITest则通过录制操作步骤生成代码，然后再执行可以起到自动化测试界面的效果。

UnitTest的使用

如何写一个简单的验证api的单元测试
创建一个single view application的项目如下图所示




Unit Tests和UI Tests都是默认选中的，然后再项目中找到XXTests.m（我的项目中是HZTestTests.m）文件;


XXTests.m详解

首先看一下文件中的几个自动生成Method； 

#import <XCTest/XCTest.h>
@interface HZTestTests : XCTestCase

@end

@implementation HZTestTests

- (void)setUp {
    [super setUp];
    // Put setup code here. This method is called before the invocation of each test method in the class.
}

- (void)tearDown {
    // Put teardown code here. This method is called after the invocation of each test method in the class.
    [super tearDown];
}

- (void)testExample {
    // This is an example of a functional test case.
    // Use XCTAssert and related functions to verify your tests produce the correct results.
}
@end
- (void)setUp

如果有一些初始代码需要在调用其他测试类的方法之前调用可以写在这个方法中；

- (void)tearDown

关于测试的方法调用完毕之后会调用

- (void)testExample

测试的业务逻辑代码可以写在这个方法中；通常是自己创建已test开头的方法类似-(void)testLogin;来验证逻辑

自定义测试方法并执行

如下图所示在左侧有空心菱形点击就可以直接执行相应的代码；没有菱形的可以command+b 一下



测试一个API的耗时时间可以直接调用measureBlock在控制台就可以看到具体的执行时间 

#pragma mark- 性能测试的耗时操作
- (void)testExcuteTime{
    // This is an example of a performance test case.
    __block NSInteger sum;
    [self measureBlock:^{
        // Put the code you want to measure the time of here.
        for (NSInteger index=0; index<10000; index++)
            sum +=index;
    }];
}

异步回调判断超时时间，创建XCTestExpectation对象，使用waitForExpectationsWithTimeout监测超时错误，使用[expectation fulfill]告知异步请求完成 

#pragma mark- 异步请求回调测试 
-(void)testAsyncRequest{
    XCTestExpectation * expectation = [self expectationWithDescription:@"test async"];
    dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
        sleep(4);
        XCTAssert(YES,"call back success!");
        [expectation fulfill];
    });
    [self waitForExpectationsWithTimeout:5 handler:^(NSError *error) {
        //Do something when time out
        NSLog(@"%@",[error localizedDescription]);
    }];
}
断言的使用
#pragma mark-  系统提供的断言 

  XCTAssert(expression, format...)
  XCTAssertTrue(expression, format...)
  XCTAssertFalse(expression, format...)
  XCTAssertEqual(expression1, expression2, format...)
  XCTAssertNotEqual(expression1, expression2, format...)
  XCTAssertEqualWithAccuracy(expression1, expression2, accuracy, format...)
  XCTAssertNotEqualWithAccuracy(expression1, expression2, accuracy, format...)
  XCTAssertNil(expression, format...)
  XCTAssertNotNil(expression, format...)   
  XCTFail(format...)

UITest的使用

HZTestUITests.m中系统会生成一些方法

#import <XCTest/XCTest.h>

@interface HZTestUITests : XCTestCase

@end

@implementation HZTestUITests

- (void)setUp {
    [super setUp];

    // Put setup code here. This method is called before the invocation of each test method in the class.

    // In UI tests it is usually best to stop immediately when a failure occurs.
    self.continueAfterFailure = NO;
    // UI tests must launch the application that they test. Doing this in setup will make sure it happens for each test method.
    [[[XCUIApplication alloc] init] launch];

    // In UI tests it’s important to set the initial state - such as interface orientation - required for your tests before they run. The setUp method is a good place to do this.
}

- (void)tearDown {
    // Put teardown code here. This method is called after the invocation of each test method in the class.
    [super tearDown];
}

- (void)testExample {
    // Use recording to get started writing UI tests.
    // Use XCTAssert and related functions to verify your tests produce the correct results.
}

如何使用UITest

  • 定义一个跳转场景viewController中点击一个button跳转到fristViewController
  • 在HZTestUITests.m定义一个-(void)testUI{}的方法；
  • 将光标放在 -(void)testUI{}的大括号中；点击下边的红色record；
  • 根据你的操作 系统会自动生成代码

下面是系统自动生成的代码详细 

  -(void)testUI{
 XCUIApplication *app = [[XCUIApplication alloc] init];
 XCUIElement *nextvcButton = app.buttons[@"nextVC"];
 [nextvcButton tap];

 XCUIElement *backButton = [[[app.navigationBars[@"HZFristView"] childrenMatchingType:XCUIElementTypeButton] matchingIdentifier:@"Back"] elementBoundByIndex:0];
 [backButton tap];
 [nextvcButton tap];
 [backButton tap];
}

执行自动UI跳转可以点击左侧的空心菱形进行操作！
