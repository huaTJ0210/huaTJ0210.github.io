Effective Objective-C 2.0（一）

发表于 2016-01-04 | 分类于 Objective-C

Objective-C是在C语言的基础上添加了面向对象的功能，它采用的是消息结构而非其他面向对象语言的函数调用；消息结构无论是否为多态都在运行时去查找相应的方法，而函数调用式的语言则由编译器决定，调用的函数是多态则按照“虚方法表”来查找到底执行那个函数。

Objective-C主要是基于runtime，它的面向对象特性所需要的全部数据结构及函数都是在runtime中的。

Objective-C语言的指针是用来指示对象的声明如下对象： 

NSString *str1 = @"the string";
NSString *str2 = str1;

如上代码会在栈内存中开辟2块区域用于存放str1、str2（32位系统是4个字节存储，64位系统是8个字节存储），在堆内存中开辟1块区域存放NSString类型的数据；栈中存放的对象不需要对内存进行管理，由系统自动在栈弹出时清理，而堆中存放的对象则由开发者通过引用计数的机制进行管理；

在类的头文件中减少引用其他类的头文件

在类的头文件中如果要使用其他类的信息可以使用向前声明的方式告诉编译器；例如在Person类的头文件中使用Employer类；可以采用@class Employer;这样的好处是可以节省编译的时间；同时向前声明也解决两个类相互引用的问题；

多用字面量少用与之等价的方法

  • 字符串字面量

    NSString* someString = @"some string";

  • 字面数值(整数、浮点数、布尔值封装为 Objective-C对象)

    NSNumber* intNumber = @1;
    NSNumber* floatNumber = @2.5f;

  • 字面量数组: 可以通过取下标(subscripting)的方式获取制定的元素

    NSArray* animals = @[@"1",@"2",@"3"];//要检测每一个数据项是否为空值
    NSString* dog = animals[1];

  • 字面量字典

    NSDictionary* personData = @{@"1":"a",@"2":"b"};//键值和value都要检测

多用类型常量，少用#define预处理指令

#define ANIMATION_DURATION 0.3表示动画的持续时间没有表示出类型信息，可采用static const NSTimeInterval kAnimationDuration = 0.3表示；
这里使用static修饰表示定义的变量只在当前编译单元(可以理解为当前类的实现文件)中可见，如果不加static编译器则会添加一个”外部符号”，如果其他文件中定义相同的变量则会抛出错误信息； 

duplicate symbol _kAnimationDuration in : 
xx.o
yy.o

当出现我们要公开某个常量的时候，可以在头文件中做如下定义： 

// in the header file
extern NSString *const EOCStringConstant;
// in the implementation file
NSString *const EOCStringConstant = @"string";

需要注意的是const修饰符在常量类型中的位置。常量定义应该是从右向左读，EOCStringConstant就是“一个常量，而这个常量是指针，指向NSString对象”，此变量的存储放在全局区中。

使用枚举来列举状态、选项、状态码

  • NS_ENUM 定义普通枚举类型 

    typedef NS_ENUM(NSInteger,HTTPMethod){
            GET,
            POST,
            DELETE
 };

  • switch 语句不要加default分支，这样增加新的枚举值的时候编译器才能提醒开发者
