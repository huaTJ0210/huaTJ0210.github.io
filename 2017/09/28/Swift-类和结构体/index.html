Swift-类和结构体

发表于 2017-09-28 | 分类于 iOS

Swift中结构体和类的相同点

  • 均可以定义属性和方法
  • 使用构造函数来初始化成员变量
  • 使用扩展来增加自身功能
  • 遵守协议

Swift中结构体和类的不同点

  • 结构体是值类型，赋值时是值拷贝
  • 类是引用类型，赋值只是指针拷贝，但指向的还是同一块内存地址
  • 类能够通过继承扩展自身功能，结构体不可以

  • 结构体有默认的初始化每一个成员变量的初始化函数，类没有

Swift的基本数据类型多是值类型

String Array Dictionary以及基本的整型、浮点型、布尔型都是值类型，底层都是结构体实现的

恒等运算符

如果希望检测两个常量或者变量是否引用同一个实例可以使用恒等运算符：

  • 等价于（ === ）
  • 不等价于 ( !== )

结构体和类的选择原则

在实际项目中如何选择结构体或类的使用呢，可以将下列情况作为选择结构体的依据：

  • 定义的数据结构是用来封装少量相关简单数据值的
  • 数据结构实例被赋值或传递，封装的数据只是被拷贝而不是引用传递
  • 数据结构中存储的也是值类型的属性
  • 数据结构不需要去继承另一个既有类型的属性或者行为

下标（subscript）

class Subscripts: NSObject {
   // 下标
    func testFunc(){
        let threeTimeTable = TimesTable(multiplier: 3)
        print(threeTimeTable[6])
    }
}

struct TimesTable {
    let multiplier:Int
    subscript(index:Int)->Int{
      return multiplier * index
    }
}
